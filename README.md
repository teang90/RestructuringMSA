# 프로젝트: 레거시 플랫폼의 MSA 전환

### 1. 프로젝트 소개

통신사 특정 도메인에서 운영 중이던 **레거시 플랫폼 4종을 통합하여 MSA 아키텍처로 전환하는 프로젝트**를 수행했습니다.  
기존 시스템은 단일 서비스 구조와 복잡한 연계 구조로 인해 운영 및 유지보수 비용이 지속적으로 증가하고 있었으며,  
이를 해결하기 위해 **서비스 단위 분리 및 이벤트 기반 MSA 구조**로 재설계했습니다.

본 포트폴리오에서는 **제가 담당했던 모듈을 중심으로 아키텍처를 설명**합니다.

---

### 2. 기대 효과

- **운영 리소스 감소**  
  - VM 장비 수 약 **160대 → 30대 수준으로 축소**
  - 불필요한 시스템 중복 제거 및 서비스 통합 효과

- **유지보수 포인트 감소**  
  - 레거시 시스템 통합 과정에서
    - 중복 코드 제거
    - 불필요한 구조 및 DB 이력 정리
  - 코드 복잡도 및 운영 리스크 감소

---

### 3. 아키텍처

- MSA 기반 서비스 분리 구조
- 이벤트 기반 비동기 처리
- Hexagonal Architecture(Ports & Adapters) 적용
- Core Domain과 외부 시스템(REST, MQ, DB, Cache) 간 의존성 분리

### 4. 기술 스택

- **Language / Runtime**
  - JDK 17

- **Framework / Network**
  - Spring Boot 3.x
  - Netty 4.x

- **Messaging / Streaming**
  - Kafka
  - Apache Artemis

- **Data / Cache**
  - Redis
  - MongoDB
  - Altibase

---

### 5. 담당 모듈의 핵심 기능

- 외부 시스템(REST, Kafka, TCP)으로부터 유입되는 요청을   **내부 트랜잭션 처리 규격에 맞게 파싱 및 변환**
변환된 요청을 **Biz Core로 전달하여 트랜잭션 처리 요청**
- MQ를 활용한 **비동기 트랜잭션 처리 구조 구현**
- Redis를 이용한
  - 트랜잭션 이력 관리
  - 트랜잭션 상태 관리
- Redis 캐시를 활용한
  - **유사 트랜잭션 재처리**
  - 중복 요청에 대한 처리 효율 향상

---

### 6. 프로젝트 요약

레거시 중심의 복잡한 구조를 MSA 기반으로 재설계하여  
**운영 리소스와 유지보수 비용을 크게 절감**하였으며,  
이벤트 기반 비동기 처리와 Redis 캐시 활용을 통해  
**트랜잭션 처리 안정성과 확장성을 확보한 프로젝트입니다.**
